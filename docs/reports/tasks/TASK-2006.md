# Golden Dataset Comparison

The parser was successfully validated against `firewall_connection_allowed.json` and `firewall_connection_blocked.json`.

**Differences Documented:**
1. **Timestamp Origination:** The golden dataset `raw_event` string (`10.0.5.15,203.0.113.88,50124,443,tcp,allow,Outbound_HTTPS`) lacks any temporal data. Thus, the parser defaults to generating a current UTC ISO-8601 string if the payload doesn't arrive pre-wrapped in JSON with a `timestamp` field.
2. **Metadata Context:** The golden datasets provide additional fields such as `bytes_sent` and `bytes_received` via metadata. Our implementation accepts arbitrary contextual attributes if they are forwarded via a JSON payload.

# Errors Found

- **Entity Validation Crash:** During initial testing, providing an empty string or only a port mapping to the `actor` and `target` Entities triggered a strict Pydantic `ValidationError`. This was resolved by conditionally verifying that at least one identifying field (id, name, username, hostname, ip) exists prior to Entity instantiation.

# Problems / Observations

1. **Protocol Ambiguity:** The firewall mappings copy raw protocol strings (e.g., `tcp`, `udp`, `icmp`) directly into `metadata.protocol`. Without an IANA protocol number validation layer, it's highly vulnerable to non-standard vendor syntax.
2. **NAT Visibility Risks:** The parser maps `src_ip` and `dst_ip` natively. It does not account for Network Address Translation (Pre-NAT/Post-NAT addressing). This restricts external network attribution capabilities.
3. **Artifact Generation Logic:** Currently, the parser only translates the `target.ip` into a searchable `Artifact`. If the firewall drops an inbound scan from an attacker, the `source.ip` (the malicious entity) is not natively extracted as an IOC, limiting threat intelligence matching.

# Recommendations

- Implement an advanced IP validation step to tag IPs as RFC-1918 (Private) vs Public, allowing dynamic bidirectional artifact generation.
- Ensure the ingestion pipeline (Fluentbit/Logstash) injects the syslog receipt timestamp into the `RawLog` JSON wrapper before forwarding it to the Batch Processor.

# Sprint Impact

The `FirewallParser` introduces critical network telemetry into ASTRA. Combining this with the newly implemented `WindowsParser`, the platform is now uniquely positioned to perform complex host-to-network correlations.
