# Mandatory Output Sections

Every final response or Phase Report produced by an agent MUST include the following structured sections:

### 1. Findings
Summary of what was discovered during Context Loading and Repository Review.

### 2. Root Cause
If applicable (e.g., debugging), the definitive source of the issue.

### 3. Plan
The agreed-upon implementation steps.

### 4. Changes
Specific files modified and logic altered.

### 5. Validation
Evidence of validation, specifically citing CI/CD or GitHub Actions success if applicable.

### 6. Documentation Updates
List of documentation files created or modified.

### 7. Risks
Any residual risks or considerations for the future.

### 8. Recommendations
Next steps or suggestions for subsequent phases.
